# App Development

Ties the other tracks together by shipping something usable end-to-end:
a backend API, a frontend UI, and a capstone that plugs in a model or
pipeline from an earlier track.

1. [`backend-api-fastapi/`](backend-api-fastapi/) — a real backend: routing, DB, auth basics.
2. [`frontend-basics-react/`](frontend-basics-react/) — components, state, calling an API.
3. [`full-stack-capstone-project/`](full-stack-capstone-project/) — combine backend + frontend + a model from another track.

Do this track last — it's most valuable once you have something (a model,
a pipeline) from an earlier track worth shipping.
