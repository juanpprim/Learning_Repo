A*: goal-directed pathfinding

A* explores the map as a graph, always expanding the node with the lowest f = g + h, where g is the exact cost from the start and h is a heuristic estimate to the goal (Manhattan distance on grids). The heuristic is the magic: it biases the search toward the goal, so A* explores a narrow corridor where Dijkstra (h=0) floods in all directions. If h never overestimates (admissible), the path found is optimal. Data structures: a priority queue (heapq) for the frontier, plus a came_from map to reconstruct the path.

push start to frontier
pop lowest f = g + h
expand walkable neighbors
goal popped → walk came_from back
The heuristic steers; the queue guarantees best-first order

Grids vs. navmeshes

Grid

  • Map = 2D array; each cell a node
  • Trivial to build — perfect for tile games & learning
  • Memory scales with area; paths look blocky without smoothing

Navmesh

  • Walkable space = few convex polygons
  • Nodes are polygons → tiny graph, natural any-angle paths
  • What 3D engines (Godot, Unity, Unreal) use; harder to generate
Same A* algorithm on both — only the graph changes

Noise: structured randomness

Pure randomness looks like static; terrain needs values that vary smoothly — neighboring points should be similar. Gradient noise (Perlin/Simplex) provides exactly that: a deterministic function noise(x, y) → [-1, 1] that is smooth, seedable, and evaluable at any point independently (so infinite worlds generate lazily). Layering octaves — adding the noise to itself at doubling frequencies and halving amplitudes (fractal noise) — produces the mix of large landmasses and small detail that reads as natural terrain. Thresholding the values assigns biomes: water below 0.3, sand to 0.4, grass, rock, snow.

1 octave — smooth blobs
3 octaves — hills + detail
5 octaves — natural-looking terrain
Each octave adds finer structure at lower amplitude

Cellular automata: growing caves

A completely different generator, ideal for organic cave systems: fill a grid randomly (~45% walls), then repeatedly apply one local rule to every cell — "become a wall if 5+ of your 8 neighbors are walls". Each smoothing pass rounds off noise; 4–5 passes yield connected caverns with natural-looking walls. Global structure emerges from a purely local rule — the same phenomenon as Conway's Game of Life. Post-processing (flood fill to find caverns, connect or discard small ones) turns raw output into playable maps.

NPC brains: finite state machines

Give an NPC a small set of states — PATROL, CHASE, ATTACK — each with its own per-tick behavior, and explicit transition conditions (player within 10 tiles: PATROL→CHASE; lost sight for 3s: CHASE→PATROL). FSMs are easy to write, debug (log the state!) and tune, which is why they still power most game AI. Their limit is state explosion as behaviors multiply — the industry's next step is behavior trees (hierarchical, composable selectors/sequences), worth knowing by name. Note this is the third appearance of FSMs in this repo: game screens, NPC brains — same pattern.
