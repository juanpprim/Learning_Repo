# Data Engineering at Scale

Most real-world data science problems are data engineering problems first.
This track builds the scale/tooling side: distributed compute, streaming,
orchestration, transformation, and cloud warehousing.

1. [`spark-fundamentals/`](spark-fundamentals/) — distributed DataFrames, joins, partitioning.
2. [`streaming-kafka/`](streaming-kafka/) — event streams, producers/consumers, windowing.
3. [`orchestration-airflow/`](orchestration-airflow/) — scheduling and dependency graphs for pipelines.
4. [`modern-data-stack-dbt/`](modern-data-stack-dbt/) — SQL-first transformations, testing, lineage.
5. [`cloud-data-warehouses/`](cloud-data-warehouses/) — BigQuery/Snowflake-style warehouse patterns.

Each subtopic is self-contained (README + starter notebook + requirements.txt).
